Storage conventions — clean up the inconsistency

Currently: mobile parser → _assets/, ingest agent → Inbox/_processed/. Needs one source of truth.

Where images live

_assets/
├── <session_id_1>/
├── page-01.jpg
├── page-02.jpg
└── manifest.json
└── <session_id_2>/
├── page-01.jpg
└── ...
← All original images, immutable, per-session folders

What Inbox/_processed/ should contain

Only non-image raw sources (text notes, PDF excerpts, markdown files). Images go to _assets/ exclusively. This matches the "raw-guard.sh" invariant — immutability is per-pipeline, not per-directory.

Frontmatter image_source field

Always points to the session folder in _assets/:

image_source: _assets/session-abc123/

If a single image is referenced (not a session), it points to the specific file:

image_source: _assets/session-abc123/page-01.jpg

Where .canvas files live

Alongside their .md note in Knowledge/<domain>/. Same slug, different extension.

Knowledge/cs/
├── binary-search-trees.md
├── binary-search-trees.canvas
├── sorting-algorithms.md
├── sorting-algorithms.canvas

Canvas files are NOT registered in _moc-registry. The .md is the navigation entry point.
