## Process
1. Read manifest.json from the session directory.
2. For each image in the session, classify content type:
├── Math → existing LaTeX transcription (unchanged)
├── Mind map / spatial diagram → invoke akasha-diagram-parser for Canvas
├── Flowchart / process / algorithm → invoke akasha-diagram-parser for Mermaid
├── System architecture → invoke akasha-diagram-parser for Canvas
├── Sketch / annotation → describe + [[image link]] (fallback)
└── Mixed (math + diagram) → math note with ![[diagram.canvas]] embed
3. New: diagram processing sub-flow
When image is classified as diagram: a. Load akasha-diagram-parser skill
b. Generate diagram output (Canvas or Mermaid)
c. Validate output per format's rules
d. If low confidence → fall back to image-link note
e. Create .canvas file alongside .md note (if Canvas path)
f. Embed/link in .md note body
4. Validation:
— LaTeX validation (unchanged)
— Canvas JSON validation (ref: validation/ references)
— Mermaid syntax validation
— If validation fails: notify and stop
5. Output files:
— Math: Knowledge/<domain>/<title>.md (type: math)
— Canvas path: Knowledge/<domain>/<title>.md + .canvas
— Mermaid path: Knowledge/<domain>/<title>.md (Mermaid block in body)
— Image-link: Knowledge/<domain>/<title>.md (type: concept, [[image]] in body)
6. Updated: image_source frontmatter
All paths → _assets/<session_id>/<filename>.jpg
(was: StudyMaterials/inbox/<session_id>/)
7. Move raw images to _assets/<session_id>/ and commit.
8. Report: session_id, domain, mocs, generated files, image count.

The ingest agent (akasha-ingest) also needs minor updates — its photo-to-LaTeX sub-flow currently uses [!figure] callouts for diagrams. It should invoke the same skill instead. Storage path also needs alignment (currently archives to Inbox/_processed/ instead of _assets/).
