Error handling and edge cases

What happens when things go wrong

Can't classify the image

Image is blurry, too messy, or ambiguous. → Create image-link note with > [!warning] Could not classify. Image stored in _assets/ for manual review.

Canvas generation fails validation

Invalid JSON, <3 nodes, all text is "[?]" markers. → Fall back to image-link note. Original image stays in _assets/.

Mermaid syntax is invalid

Unclosed brackets, bad node references, unrenderable syntax. → Fall back to image-link note.

Mixed page: math + diagram

Vision LLM extracts both. Math goes in the .md body, diagram goes in a .canvas embedded via ![[.canvas]]. One note, two outputs.

Partial output (some good, some bad)

Canvas has 5 good nodes + 3 "[?]" nodes. → Keep the good nodes. Add a [?] node at the bottom noting missing content. Don't discard the whole output.

Multiple sessions for same topic

Existing note already exists. → Update existing note, append new content. Don't duplicate.
