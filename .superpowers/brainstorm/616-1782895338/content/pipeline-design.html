Pipeline Flow

One photo → classification → typed output

Modified akasha-mobile-parser
For each image in a capture session:
1. Classify content
Vision LLM decides: math / diagram / sketch / mixed
2. Route to path
├── Math
→ Existing LaTeX transcription → math note
├── Diagram (structured)
→ Transcribe to JSON Canvas 1.0 → .canvas file
├── Sketch / annotation
→ Describe in text → concept note with [[image link]]
└── Mixed content
→ Dominant type's output, secondary content embedded inline
3. Store original image
→ Moves to _assets/<session_id>/ regardless of path
4. Create output
Math
→ Knowledge/<domain>/<title>.md (type: math)
Diagram → Canvas
→ Knowledge/<domain>/<title>.canvas
Diagram → fallback
→ Knowledge/<domain>/<title>.md (type: concept)

Output types and their frontmatter

Canvas path (.canvas)
JSON Canvas 1.0 file. Text nodes with transcribed content, positioned spatially. Edges between nodes. Original photo linked as a file node inside the canvas.
Canvas is the diagram — no separate .md file needed. Canvas IS the entry point.
Fallback path (.md)
Standard concept note with: [[_assets/session-id/image.jpg]] in the body instead of an embed.
Description in the note tells you what the diagram shows. Click the link to open the image.
