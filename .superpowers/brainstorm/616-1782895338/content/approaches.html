Three Approaches: How to capture drawn diagrams

Each has different failure modes. Which direction feels right?

A

Image + Rich Metadata

Low failure

  • Vision describes what it sees — almost never wrong
  • Photo is embedded as ![[photo.jpg]]
  • Frontmatter tags + description = searchable

Downsides

  • Diagram is a static image — can't edit, link, or zoom nodes
  • Not deeply integrated into the knowledge graph
B

Transcribe to Obsidian Canvas

High payoff

  • Produces .canvas file — editable, zoomable in Obsidian
  • Each node is its own text — linkable to notes
  • Original image can be embedded inside the canvas

Fails more

  • Vision must parse spatial layout + handwriting + connections
  • Messy sketches → garbled output
  • More tokens per capture
C

Tiered: Try Canvas, Fall Back to Image

Best of both

  • Classifier decides: structured (mind map) → try Canvas
  • Sketchy/annotated → image + metadata immediately
  • Canvas attempt has confidence check — falls back gracefully

More complex

  • Two pipeline paths to maintain
  • Classifier can misclassify (rare but possible)
  • More tokens when it tries Canvas and fails
