Canvas vs Mermaid — split by diagram type

Each format has a natural fit. The classifier picks the right one.

Expanded classification tree
Image → Classify:
├── Math
→ LaTeX + markdown → .md note (existing path)
├── Mind map / spatial diagram
→ Canvas .canvas file + .md note with ![[.canvas]]
├── Flowchart / process / algorithm
→ Mermaid block inside .md note
├── System architecture
→ Canvas (too complex for Mermaid's constraints)
└── Sketch / annotation
→ image-link note ([[_assets/image.jpg]])

Trade-offs at a glance

Canvas
Best for: spatial/free-form
  • Free positioning = preserves original layout
  • Can embed image file nodes
  • Group zones, colored edges, labels
  • Separate .canvas file needed
  • Not inline-renderable in markdown
Mermaid
Best for: structured/sequential
  • Renders inline in .md — no extra file
  • Easy to edit text
  • Good for flowcharts, sequence diagrams
  • No free positioning — auto-layout only
  • Limited colors/groups/images
  • Poor fit for mind maps

Failure mitigation — both paths

Canvas validation gates
  • Parse valid JSON
  • ≥3 nodes with non-empty text
  • Edge IDs reference existing nodes
  • Coordinates within reasonable bounds (0–4000)
  • Few "[?]" markers → confidence drop
  • Low confidence → image-link note
Mermaid validation gates
  • Valid Mermaid syntax (no unclosed quotes/brackets)
  • Nodes have IDs and text labels
  • Edges connect existing nodes
  • No unsupported node shapes/types
  • Low confidence → Canvas try → image-link fallback
Key mitigation: If either format produces low-confidence output, the note still gets written — but with a > [!warning] callout at the top saying the transcription may be inaccurate and linking to the original image.
